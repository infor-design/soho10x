BREAKING CHANGES

inforFormButton and inforTextButton deprecated

inforFormButton and inforTextButton are javascript widgets, since they now work as css only. Just remove all calls such as $(something).inforFormButton() and $(something).inforTextButton()

  
     
     
 

Lazy loading tree grid

The tree grid now supports lazy loading as well as sorting. However, in order to do this you need to change how you pass the data in. The important columns to add are:

depth: This was the indent and should be a search and replace. Set this to the level of the node.
parent: This is the id (idProperty value). Of the parent record.
isParent: If this is a parent set this to true (now required) this is especially important for lazy loading so that it is known that there are children.
children: Not used yet but can be added. Add the id's of all the children for the parent records.

Records now do not need to be in order and sorting will not hang, but a correction will follow on sorting within levels.

Fieldset markup

The previous fieldset markup was incorrect, as the legend element was not used correctly. This was adjusted for accessibility (508). The new mark-up affects the expand button and fieldset label elements

<fieldset class="inforFieldSet">
   <legend>
      <span class="inforFieldSetLabel">The Label</span>    
   </legend>  
   <button type="button" class="inforExpandButton open"></button>  
   <div class="content">  
      <p>Lorem ipsum dolor sit amet,</p>
   </div>
</fieldset>
