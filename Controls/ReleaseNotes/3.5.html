HTML controls release 3.5 is now available

Featuring new controls, and new SoHo guidelines.

The Infor HTML controls release 3.5 is now available now to all users. This is mainly a bug fix release with only a few new features. As well as a few additional improvements for accessibility.

Refer to this release as your continuing resource for understanding and implementing the most current SoHo user interface.

NOTABLE CHANGES

Error Field Positioning Change

The validation errors now appear on the bottom (like the will in the next version), this is to improve positioning in edge cases and mobile support.

 
     
     
 

Rich Text Editor

The editor now has a static toolbar.

 
     
     

Context Menu Positioning Improvements

Added fixes for menu positional edge cases with context menu and its releated controls.

     
     
 

Context Menu Changes

The Context menu is renamed to popupmenu. While most of the old options and config should work it is reccomended to use the new code. See Main example. Options iFrameFix and backgroundIframe are now auto detect so should not be needed in code. Callback is depricated use events like selected or click on the anchor tags instead. Also default is now click not rightClick. Now just two options: menuId: null, //Menu Id or looks if the next element is a popupmenu. trigger: 'click'
