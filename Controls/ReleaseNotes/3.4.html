NOTABLE CHANGES

New Coding Standards

We've landed on coding standards for Sass/Node.js and JS projects. All code will be changed finalized in 4.0 to drop the infor extension and follow the following guidelines http://git.infor.com/projects/SHARED/repos/coding-standards/browse. All code in 4.0 and above must pass jsHint.

 
     
     
 

inforMessage Dialog is now modal() or message()

The inforMessageDialog plugin was rewritten for 4.0 and ported back for better accessibility and responsiveness. For easier compatibility inforMessageDialog is mapped to .message() automatically however it should be considered depricated and will eventually be removed.

inforMessageDialog - General -> Is now .modal() see examples for the new markup changes. inforMessageDialog - (Alert, Message, Info) -> Is now .message() - api is the same.

It was not possible to map calls in buttons like $(this).inforDialog('close') these MUST be changed to $(this).modal('close')

 
     
     

Module Buttons need extra elements

The module buttons (close/settings ect) now need an i element added fx

<button title="Settings" type="button" class="inforModuleSettingsButton">
<i>Settings</i>
</button>

     
     
 

Context Menu Changes

The Context menu is renamed to popupmenu. While most of the old options and config should work it is reccomended to use the new code. See Main example. Options iFrameFix and backgroundIframe are now auto detect so should not be needed in code. Callback is depricated use events like selected or click on the anchor tags instead. Also default is now click not rightClick. Now just two options: menuId: null, //Menu Id or looks if the next element is a popupmenu. trigger: 'click'

setValue Function Depricated

the $(thing).setValue() function has been depricated and is of limited use. Replace your calls with standard jquery functions to set values for example:

$('#selectId').val('Option 1').trigger('updated'); //Set Drop Down Value $('#checkboxId').prop('checked', 'true'); //Set Checkbox Valu

 
     
     
     
 

Validation Api Changes

Some changes to the Validation API however functions have been restored for backwards compatibility. When possible migrate your

  • data-validate attribute is now data-validation
  • validationMessage("show", message, true) is now addError(...)
  • validationMessage("remove") is now removeError()
  • setupValidation(function) is now .validate().on('validated', fn);
  • *addRule(ruleName, {...})* is now $.fn.validation.rules[ruleName] = {...}
